# 重复购买预测实验报告

## 1. 项目概述

### 1.1 问题背景
重复购买预测是电商领域的经典问题，目标是识别哪些用户会对某个商家产生重复购买行为。这个问题具有重要的商业价值：
- **精准营销**：针对高复购倾向用户进行定向推广，提高营销ROI
- **库存优化**：预测复购需求，优化库存配置
- **用户运营**：识别高价值用户，制定差异化运营策略
- **商家服务**：帮助商家了解自身吸引力，改进服务

### 1.2 问题特点分析
这是一个典型的**二分类问题**，但具有以下特殊性：

**样本不平衡**：实际场景中重复购买用户占比较低（通常5-15%），需要选择适合不平衡数据的评估指标和算法。

**时间依赖性**：用户行为具有时序特征，最近的行为对预测更重要。

**多维度关联**：预测结果受用户特征、商家特征、用户-商家交互历史三方面共同影响。

### 1.3 数据集说明
- **训练集**：260,864条标注样本
- **测试集**：261,477条待预测样本
- **用户行为日志**：记录用户在平台上的浏览、加购、收藏、购买等行为
- **用户画像**：年龄段、性别等人口统计学信息

### 1.4 评估指标选择
采用 **AUC (Area Under ROC Curve)** 作为主要评估指标，原因如下：

**适合不平衡数据**：AUC关注排序质量而非绝对预测值，不受正负样本比例影响

**业务契合度高**：在实际应用中，我们通常根据预测概率排序选择Top-K用户进行营销，AUC正好衡量了这种排序能力

**阈值无关性**：不需要预先设定分类阈值，更客观地评估模型性能

## 2. 特征工程原理

特征工程是本项目的核心，遵循"全局画像 + 局部交互"的设计理念。

### 2.1 特征体系设计思路

#### 2.1.1 用户维度特征（15个）

**设计原理**：刻画用户的整体购买习惯和偏好

**行为活跃度特征**
- **总行为次数**：反映用户活跃程度，活跃用户更可能重复购买
- **各类行为计数**：区分浏览、加购、收藏、购买四种行为，捕捉用户的决策路径
- **活跃天数**：衡量用户粘性，长期活跃用户更稳定

**购买转化特征**
- **购买次数与购买率**：直接反映用户的购买意愿强度
- **日均行为次数**：区分"深度用户"和"偶尔访问用户"

**多样性特征**
- **访问商家数、品牌数、品类数**：反映用户的探索行为
- 高多样性用户可能对单一商家忠诚度较低
- 低多样性用户可能已找到心仪商家，复购概率更高

**时间特征**
- **首次/最后行为时间**：捕捉用户生命周期阶段
- 新用户可能处于探索期，老用户可能进入稳定期

**人口统计学特征**
- **年龄段、性别**：不同人群的购买习惯差异显著

#### 2.1.2 商家维度特征（15个）

**设计原理**：刻画商家的吸引力和经营质量

**流量质量特征**
- **访问用户数与总行为数**：衡量商家的流量规模
- **人均行为次数**：流量质量指标，高人均行为说明用户对商家兴趣浓厚

**转化能力特征**
- **购买用户数与购买转化率**：核心指标，直接反映商家吸引力
- 高转化率商家更容易产生重复购买

**商品丰富度特征**
- **商品数、品牌数、品类数**：商品多样性影响用户选择空间
- 品类丰富的商家能满足用户多样化需求

**行为分布特征**
- **各类行为占比**：反映用户在该商家的决策模式
- 高收藏率可能说明用户有购买意向但尚未转化

#### 2.1.3 用户-商家交互特征（12个）

**设计原理**：这是最关键的特征类别，直接反映用户对特定商家的兴趣程度

**交互强度特征**
- **总行为次数**：用户对该商家的关注度
- **交互活跃天数**：反映关系的持续性，多次访问说明持续关注

**行为模式特征**
- **各类行为计数与占比**：
  - 浏览为主：可能只是随便看看
  - 加购/收藏为主：有购买意向但犹豫
  - 购买行为：已经产生过购买，重复购买概率大增

**转化指标**
- **购买率、加购率、收藏率**：精细刻画用户对该商家的转化漏斗
- 这些比率特征比绝对数值更稳定，泛化性更好

**多样性特征**
- **浏览商品数、品类数、品牌数**：
  - 高多样性：用户在该商家广泛浏览，说明兴趣广泛
  - 低多样性：可能已锁定特定商品，目标明确

### 2.2 特征工程的技术优化

**原始方案的问题**：
逐行遍历处理用户-商家对，对26万条训练数据需要执行26万次循环，每次都要筛选和统计，时间复杂度为O(n×m)，其中n是样本数，m是行为日志数。

**优化方案的原理**：
利用Pandas的groupby机制，一次性对整个行为日志进行分组聚合：
1. 按(user_id, merchant_id)进行分组
2. 同时计算所有需要的统计量
3. 通过merge操作批量关联特征

**优化效果**：
- 时间复杂度降低到O(m + n)
- 利用了Pandas底层的C优化和向量化运算
- 实际速度提升50-100倍

**最终特征规模**：
- 训练集：260,864 × 43（包含label）
- 测试集：261,477 × 42
- 有效预测特征：40个（排除user_id和merchant_id）

## 3. 模型原理与设计

### 3.1 算法选择：LightGBM

**为什么选择LightGBM而非其他算法？**

**相比逻辑回归**：
- 逻辑回归是线性模型，难以捕捉特征间的非线性交互
- 本问题中，用户特征、商家特征、交互特征之间存在复杂的组合关系
- 例如："高购买力用户 × 高转化率商家" 这种交互效应需要非线性模型

**相比传统GBDT**：
- LightGBM采用leaf-wise生长策略，比XGBoost的level-wise更高效
- 支持大规模数据训练，内存占用更小
- 训练速度快，便于快速实验迭代

**相比深度学习**：
- 表格数据上树模型通常优于神经网络
- 不需要复杂的特征归一化和大量调参
- 模型可解释性更强，可以分析特征重要性

### 3.2 LightGBM核心原理

**Gradient Boosting框架**：
- 采用加法模型，每次迭代训练一棵新树来拟合之前模型的残差
- 通过梯度下降优化损失函数
- 最终预测结果是所有树预测值的加权和

**Leaf-wise生长策略**：
- 每次选择增益最大的叶子节点进行分裂
- 相比level-wise（逐层生长），能更快降低损失
- 通过max_depth限制防止过拟合

**直方图算法**：
- 将连续特征离散化为直方图
- 大幅降低内存占用和计算复杂度
- 对噪声数据有一定鲁棒性

### 3.3 参数设计原理

**树结构参数**：
```
num_leaves: 40        # 叶子节点数，控制模型复杂度
max_depth: 8          # 最大深度，防止过拟合
min_child_samples: 20 # 叶子节点最小样本数，保证统计可靠性
```
这组参数平衡了模型的表达能力和泛化能力。

**学习参数**：
```
learning_rate: 0.05   # 较小的学习率配合更多迭代次数
```
小学习率能让模型更平稳地收敛，避免过度拟合训练数据。

**正则化参数**：
```
feature_fraction: 0.8  # 每次迭代随机选择80%特征
bagging_fraction: 0.8  # 每次迭代随机选择80%样本
reg_alpha: 0.1        # L1正则化
reg_lambda: 0.1       # L2正则化
```
多重正则化机制共同防止过拟合，提升泛化能力。

### 3.4 训练策略：7折交叉验证

**原理**：
- 将数据分为7份
- 轮流使用其中1份作为验证集，其余6份作为训练集
- 每折训练一个独立模型
- 最终预测时对7个模型的输出取平均

**Out-of-Fold (OOF) 预测**：
- 每个样本恰好在一个fold中作为验证集
- 汇总所有fold的验证集预测，得到全部训练数据的OOF预测
- OOF AUC是对模型泛化能力的无偏估计

**优势分析**：
1. **评估更可靠**：7次实验的平均结果，消除单次划分的偶然性
2. **充分利用数据**：每份数据都被用作训练和验证
3. **模型融合**：测试集预测时，7个模型的集成能降低方差，提升稳定性
4. **发现过拟合**：如果train/valid AUC差距大，说明过拟合严重

## 4. 实验结果分析

### 4.1 模型整体性能

**测试集预测统计**：
```
总预测样本数：261,477
预测概率范围：[0.007737, 0.709909]
预测概率均值：0.060691
预测概率中位数：0.047380
标准差：0.043685
```

**性能解读**：
- 模型成功加载了7个交叉验证训练的子模型
- 预测概率分布合理，集中在0-0.2区间
- 最大预测概率为0.71，说明模型确实识别出了高置信度样本
- 平均概率6%左右，符合实际重复购买率的数量级

#### 4.2.2 分布特征分析

**长尾分布特征明显**：
- **87.57%的样本概率低于0.1**：这些是明显的"不会重复购买"群体
- **仅1.77%的样本概率高于0.2**：高潜力用户非常稀缺
- **仅0.37%的样本概率高于0.3**：这是最核心的目标用户群

**分位数分析**：
```
25%分位数：0.034143  → 75%的样本概率 > 3.4%
50%分位数：0.047380  → 中位用户概率约4.7%
75%分位数：0.071937  → 25%的样本概率 > 7.2%
```

这个分布反映了**电商复购的真实场景**：
- 大多数用户-商家配对不会产生重复购买
- 只有少数特定的匹配才会产生复购行为
- 模型成功识别出了这种稀疏性

### 4.4 模型校准度分析

**预测概率的合理性验证**：

**均值分析**：
- 预测均值：6.07%
- 如果真实复购率在5-10%，则模型校准良好
- 说明模型没有系统性高估或低估

**分布形态**：
- 概率集中在低值区间，符合不平衡数据特征
- 存在明确的高概率样本（>0.5），说明模型有区分能力
- 没有出现异常的概率聚集，分布平滑合理

**四分位差分析**：
```
IQR = Q3 - Q1 = 0.071937 - 0.034143 = 0.037794
```
- 四分位差较小，说明大部分样本概率集中
- 但存在长尾，说明模型能识别异常值

### 4.5 特征重要性洞察

根据模型训练过程中的特征重要性分析，Top关键特征包括：

**交互特征主导**：
1. `um_total_actions`：用户对商家的总行为数
2. `um_action_2_cnt`：历史购买次数
3. `um_buy_ratio`：用户对商家的购买转化率
4. `um_active_days`：交互活跃天数

**用户特征辅助**：
1. `user_buy_cnt`：用户总购买次数
2. `user_buy_ratio`：用户整体购买率
3. `user_total_actions`：用户总活跃度

**商家特征补充**：
1. `merchant_buy_ratio`：商家整体转化率
2. `merchant_total_actions`：商家流量规模

**关键洞察**：
- **历史行为是最强信号**：有购买历史的用户，复购概率指数级提升
- **交互频次至关重要**：访问次数越多，关系越稳固
- **用户基线很重要**：经常购物的用户，在各商家复购概率都更高

### 4.7 预测结果质量评估

**优点**：
**概率分布合理**：符合业务实际，没有异常聚集
**区分度明显**：能清晰区分高、中、低潜力用户
**极值可控**：最大概率0.71，没有过度自信
**稳定性好**：7折模型ensemble降低了方差

**需要注意的点**：
**高概率样本稀缺**：仅66个样本>0.5，限制了高精度营销的规模
**阈值敏感**：从0.3到0.2阈值，样本数从980增加到4600，增长4.7倍
**冷启动问题**：对于无历史交互的用户-商家对，预测偏保守
